# Багатошаровий перцептрон (MLP)

## Опис

Multi-Layer Perceptron — основа нейронних мереж, одна з перших архітектур глибокого навчання.
У різних формах використовується в кожній нейромережі і являє собою модель, яка зв'язує кожен вхід з кожним виходом певною вагою (weight) і зміщенням (bias). Складається з кількох лінійних шарів (Fully-connected / Feed-forward / Linear), розділених нелінійною функцією активації. Має фіксовану кількість входів і виходів. Можна представити шар як набір лінійних регресій. Weights і Bias — параметри нейромережі, що змінюються під час тренування.


## Штучний нейрон

Один нейрон являє собою лінійну залежність виходу від входу. Максимально проста реалізація виглядає так:
`y = w * x`

У такій формі один нейрон може виконувати функцію конвертації валют, маси та іншого, наприклад, при `w=1000` нейрон буде перетворювати кілограми в грами. Але цей підхід має значний недолік — графік завжди проходить через 0.

Оскільки нейромережа є універсальною функцією апроксимації, цей недолік позбавляє можливості апроксимувати значну частину залежностей, наприклад, конвертацію градусів Цельсія у Фаренгейти.

Ми хочемо мати можливість перетворити будь-яке число на будь-яке інше, для цього зазвичай до нейрона додається зміщення `bias`, і повна функція виглядає так:
`y = w * x + b`

Це дасть зміщення функції по вертикалі, додаткове горизонтальне зміщення не має сенсу, оскільки лінія графіка нескінченна і її можна виразити через вертикальне зміщення.


## Лінійна регресія

У більшості задач ми маємо кілька входів, з яких хочемо отримати результат. У такому випадку модель необхідно розширити до такої, де у кожного входу буде своя вага:
`y = x(1)*w(1) + x(2)*w(2) + ... + x(n)*w(n) + b`

У кількох зміщеннях `bias` немає потреби, оскільки суму кількох зміщень можна замінити одним числом `b`.

Зручно буде виразити вхід і вагу як вектори однакової довжини та множити їх поелементно.
```python
x = [1, -1,   0.2]
w = [2,  0.1, 0.5]

b = 0.5

y = sum([x[i]*w[i] for i in range(3)]) + b
print(y)
```

Це і є лінійна регресія, далі ми з'єднаємо декілька таких щоб отримати повноцінний шар.


## Linear layer

На відміну від лінійної регресії, шар має кілька виходів, які обчислюються таким самим чином:
```python
x = [1, 2, 3]

w1 = [0.5, 0.7, 0.1]
w2 = [0.3, 0.1, 0.5]

b1 = -0.1
b2 =  0.7

y1 = sum([x[i]*w1[i] for i in range(3)]) + b1
y2 = sum([x[i]*w2[i] for i in range(3)]) + b2

print(y1, y2)
```

Ми можемо об'єднати ваги в матрицю, а зміщення — у вектор:
```python
x = [1, 2, 3]

w = [[0.5, 0.7, 0.1],
	 [0.3, 0.1, 0.5]]

b = [-0.1, 0.7]

y = [sum([x[j] * w[i][j] for j in range(3)]) + b[i] for i in range(2)]
print(y)
```

Це ідентично до матричного множення (чим воно і є). Матричне множення легко зробити паралельним і виконувати на GPU. Далі ми будемо використовувати модуль `numpy` для зменшення обсягу коду, але реалізувати нейромережу можна будь-якою мовою програмування та з використанням будь-якої технології.

Важливо зважати на розміри матриць: основна умова для матричного множення — `A[n, k] * B[k, m]`.

Для можливості пакетної обробки входів та подальшого розширення замінимо вектори на матриці розміру `[1, n]`:
```python
import numpy as np

x = np.array([[1, 2, 3]]) # [1, 3] - [розмір пакету, кількість входів]

w = np.random.randn(3, 2) # [3, 2] - [кількість входів, кількість виходів]
b = np.random.randn(1, 2) # [1, 2] - [1, кількість виходів]

#a @ b = np.matmul(a, b) = матричне множення
y = x @ w + b #[1, 2] - [розмір пакету, кількість виходів]

print(y)
```

При бажанні це можна помістити в клас:

```python
class Layer:
	def __init__(self, n_inp, n_out):
		# Така ініціалізація параметрів може прискорити подальше тренування
		self.w = np.random.randn(n_inp, n_out) / (n_inp + n_out)**0.5
		self.b = np.random.randn(    1, n_out) / (    1 + n_out)**0.5

	def forward(self, x):
		self.x = x # збережемо вхід для визначення похідної
		return x @ self.w + self.b
```

**Зверніть увагу!**
Розмір матриці `b` - `[1, кількість виходів]`, в пакетній обробці результат множення `x @ w` буде - `[розмір пакету, кількість виходів]`, numpy автоматично розширює матрицю перед додаванням створюючи копію для кожного елементу пакету, що ідентично до множення матриці одиниць на bias -  `1[розмір пакету, 1] @ bias[1, кількість виходів]`. Це важливо враховувати в звовортньому розповсюдженні помилки.


## Функція активації

Як би парадоксально це не здавалося, збільшення кількості шарів не дасть приросту точності. Кілька послідовних лінійних трансформацій можна визначити єдиною лінійною трансформацією:

```python
# 2 шари підряд [3 → 5 → 2]
y = (x @ w1[3, 5] + b1[1, 5]) @ w2[5, 2] + b2[1, 2]

# розкриємо дужки
y = x @ w1[3, 5] @ w2[5, 2] + b1[1, 5] @ w2[5, 2] + b2[1, 2]

# через асоціативність матричного множення ми можемо спочатку помножити всі параметри (w, b), отримаємо нові параметри
y = x @ w[3, 2] + b[1, 2]
```

Виходить, будь-яку кількість шарів можна звести до одного шару скомпонованих параметрів. Для ускладнення архітектури необхідно прибрати лінійність, саме це робить функція активації.

Одною з перших функцій активації була ReLU:
```python
y = max(0, x)               # python
y = np.where(x >= 0, x, 0)  # numpy
```

Похідна ReLU:
```python
dx = 1 if x >= 0 else 0     # python
dx = np.where(x >= 0, 1, 0) # numpy
```

$$ \frac{\partial y}{\partial x} = \begin{cases} 1, & \text{якщо } x \ge 0 \\ 0, & \text{якщо } x < 0 \end{cases} $$

Але при тренуванні методом градієнтного спуску (далі в статті) градієнт для від'ємних значень дорівнюватиме нулю, і навчання деяких нейронів зупиняється (проблема "dying ReLU"), тому використовується простіша, але ефективніша функція LeakyReLU:

$$y = \begin{cases} x, & \text{якщо } x \ge 0 \\ \alpha \cdot x, & \text{якщо } x < 0 \end{cases}$$
де `alpha` — це зазвичай невелике значення, наприклад `0.1`.

Похідна LeakyReLU:
$$\frac{\partial y}{\partial x} = \begin{cases} 1, & \text{якщо } x \ge 0 \\ \alpha, & \text{якщо } x < 0 \end{cases}$$

У коді:
```python
class LeakyReLU:
	def __init__(self, alpha):
		self.alpha = alpha

	def forward(self, x):
		self.x = x
		return np.where(x >= 0, x, self.alpha * x)

	def backward(self, grad):
		return grad * np.where(self.x >= 0, 1, self.alpha)
```

Тепер ми можемо з'єднувати шари послідовно:
```python
h = layer1.forward(x)
h = relu1.forward(h)
h = layer2.forward(h)
y = relu2.forward(h)
```

## Зворотнє розповсюдження помилки. Функція втрат

Тренування також називають оптимізацією моделі/параметрів. На сьогодні більшість оптимізаторів базуються на градієнтному спуску. Для цього алгоритму спочатку необхідно визначити градієнт, що є набором похідних функції втрат (Loss) для кожного параметра.

Визначимо втрату (помилку моделі). Існує багато способів це зробити, але ми розглянемо популярні для регресії: MSE Loss та MAE Loss.

**MSE (Mean Squared Error)** — середнє значення квадрата різниці між виходом та цільовим значенням мережі, що визначається як:
```python
#n = y.size = target.size
loss = np.sum((y - target)**2)/n
```

Похідна:
$$\frac{\partial L}{\partial y_i} = \frac{2}{n} (y_i - \text{target}_i)$$
$$\frac{\partial L}{\partial t_i} = \frac{2}{n} (\text{target}_i - y_i)$$

```python
class MSE:
	@staticmethod
	def forward(y, target):
		assert y.shape == target.shape
		return np.sum((y - target)**2)/y.size

	@staticmethod
	def backward(y, target):
		assert y.shape == target.shape
		return 2/y.size * (y - target)
```
В даній реалізації ми повертаємо градієнт тільки для `y`, але в задачах з використанням embedding-моделі ми повинні тренувати не тільки модель на перетворенні входів датасету, а й перетворенні бажаних виходів, тому для деяких задач нам потрібно два градієнти.


**MAE (Mean Absolute Error)** — середнє значення абсолютної різниці між виходом мережі та цільовим значенням:
```python
loss = np.sum(abs(y - target))/n
```
Похідна:
$$\frac{\partial L}{\partial y_i} = \frac{\text{sign}(y_i - \text{target}_i)}{n}$$
$$\frac{\partial L}{\partial t_i} = \frac{\text{sign}(\text{target}_i - y_i)}{n}$$


## Похідні для Layer

Оскільки нам необхідна похідна для параметрів `dL/dw` і `dL/db`, ми визначимо її зворотним розповсюдженням, множачи похідну за виходом на похідну виходу за цим параметром. Також визначимо похідну для входу, щоб розповсюдити її в попередній шар і до початку моделі. Ми можемо множити похідні завдяки правилу ланцюга:
$$\frac{\partial L}{\partial w} = \frac{\partial L}{\partial y} \cdot \frac{\partial y}{\partial w}$$

Найкраще і найпростіше буде реалізувати це як зворотній прохід для кожного шару, розберемо на прикладі нашого Linear:
```python
# Forward
y = x * w + b

grad = MSE.backward(y, target) # dL/dy

grad_x = grad * w # dL/dy * dy/dx = dL/dx
grad_w = grad * x # dL/dy * dy/dw = dL/dw
grad_b = grad * 1 # dL/dy * dy/db = dL/db
```

У випадку множення матриць нам треба узгодити розміри. Оскільки ми проходимо в зворотному напрямку, треба транспонувати матриці по правилу:
```
y = a @ b
g = dL/dy

dL/da = g   @ b.T
dL/db = a.T @ g
```

Використовуючи це правило, зворотний прохід для `Layer` виглядатиме так:
```python
	#...
	def backward(self, grad)
		# grad[batch size, outputs]

		self.w_grad = self.x.T @ grad
		self.b_grad = np.sum(grad, axis=0, keepdims=True) # аналог множення одиниць[1, bs] @ grad[bs, out] / зворотня дія до дублювання bias для кожного елементу batch в прямому проході

		return grad @ self.w.T # x_grad
	#...
```

Для визначення формул інших типів шарів корисно використовувати таблицю похідних.
Також ми можемо отримати похідну для самого входу, що буде корисно у випадку використання embedding-шару для інтерпретації значень в деяких задачах.

Визначимо градієнт для всіх параметрів розповсюдивши помилку в зворотньому порядку:
```python
grad = MSE.backward(y, target)

grad = relu2.backward(grad)
grad = layer2.backward(grad)
grad = relu1.backward(grad)
grad = layer1.backward(grad)
```

## Градієнтний спуск. Чому похідна?

Уявимо, що в нас є функція `y = x` і ми хочемо зменшити `y`, тоді нам треба зменшити `x`.
Для функції `y = -x` потрібно зробити зворотне, тобто знак впливає на необхідну зміну.
Цей знак є знаком похідної, а коефіцієнт біля `x` і є похідною `dy/dx`.
Якщо ми додамо до значення його похідну, результат зросте, а при відніманні — навпаки. Саме цього ми і хочемо досягти — зменшити значення функції втрат (Loss).

У класичному градієнтному спуску ми будемо робити невеликий крок у напрямку, протилежному градієнту:
```
w -= grad_w * learning_rate
b -= grad_b * learning_rate
```

Для класу `Layer`:
```python
	# ...
	def step(self, learning_rate):
		self.w -= self.grad_w * learning_rate
		self.b -= self.grad_b * learning_rate
	# ...
```

Learning Rate — швидкість навчання, більша швидкість буде менш точною, оскільки похідна має сенс тільки в одній точці, але дуже низькі значення не є оптимальними через проблему локального мінімуму.
Цей крок треба повторювати багато разів до сходження моделі.


## Клас моделі і цикл тренування

Створимо клас `Sequential` для створення і використання лінійної моделі:
```python
class Sequential:
	def __init__(self, *args):
		self.layers = args

	def forward(self, x):
		for layer in self.layers:
			x = layer.forward(x)
		return x

	def backward(self, grad):
		for layer in reversed(self.layers):
			grad = layer.backward(grad)
		return grad

	def step(self, lr=1e-3):
		for layer in self.layers:
			if hasattr(layer, 'step'): # функція активації LeakyReLU не має параметрів
				layer.step(lr)
```

Створюємо модель і цикл тренування:
```python
x = np.array([[0, 0], [0, 1], [1, 0], [1, 1]])
target = np.array([[0], [1], [1], [0]]) # задача апроксимувати XOR

model = Sequential(
	Layer(2, 5),
	LeakyReLU(0.1),
	Layer(5, 1),
	LeakyReLU(0.1)
)

steps = 3000
lr = 0.01

for i in range(steps):
	y = model.forward(x)

	loss = MSE.forward(y, target)
	grad = MSE.backward(y, target)

	model.backward(grad)
	model.step(lr)

	print(loss)

print(y)
```


## Ініціалізація параметрів

Великі значення параметрів не бажані для градієнтного спуску. Оскільки вхід проходить через кожен шар, число буде або збільшуватись або зменшуватись, в зворотньому проході ми будемо множити градієнт на той самий коефіцієнт і градієнт буде затухати або вибухати йдучи до першого шару. В такому випадку мережа буде вчитись нестабільно і не рівномірно. Коефіцієнти, що сильно відходять від 1 будуть усилювати цей ефект. 

Через властивість матричного множення кількість входів шару впливає на діапазон значень виходу - 2 входи з коефіцієнтами ваги 1 дадуть вихід 2, 700 - дадуть 700 і функція втрат видасть занадто високий градієнт.

Існує багато методів ініціалізації параметрів, що дозволяють частково уникнути проблем з градієнтом.
Наприклад семплінг з рівномірного розпреділення в діапазоні n:
```python
n = 6**0.5 / (n_inp + n_out)**0.5
w = np.random.uniform(-n, n, (n_inp, n_out))

n = 6**0.5 / (1 + n_out)**0.5
b = np.random.uniform(-n, n, (1, n_out))
```
